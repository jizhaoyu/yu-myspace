# Java + Krema 版 Fangyu Code 阶段设计（OpenCode 单引擎）

目标：在现有 `Java + Spring Boot + Krema + React` 项目中，复刻 Fangyu Code 的核心能力，保留当前架构，不走 Rust/Tauri 迁移路线。

本设计已按当前代码结构优化，核心原则是“先收敛引擎，再补强主链，再做 Skills/MCP”。

## 全局约束

- 保持技术栈不变：`Java + Spring Boot + Krema + React`
- 不推翻现有目录与分层
- 业务逻辑放 Java service，`@KremaCommand` 保持薄
- 前端统一通过 `desktopApi` / `desktopEvents` 访问桌面桥
- 先做可用闭环，再做高级能力
- 每阶段完成后仅输出该阶段结果，并询问是否继续下一阶段

## 阶段 0：引擎收敛（最高优先级） ✅ 完成

先将系统收敛为 **OpenCode 单引擎**，避免后续 Skills/MCP 反复返工。

1. **后端收敛**
   - `AiEngineKind` 仅保留 `OPENCODE` ✅
   - `FangyuProperties.Engines` 仅保留 `opencode` ✅
   - `application.yml` 删除 `claude/codex/gemini` 配置块 ✅
   - 默认引擎和 fallback 统一为 `OPENCODE` ✅
2. **引擎实现收敛**
   - 保留/新增 `OpenCodeEngine`（CLI 型） ✅
   - 已下线 `ClaudeEngine`、`GeminiEngine` ✅
   - `CodexEngine` 代码已保留但未被 Spring 注入，待彻底移除（阶段 1）
   - `CostTrackerService` 仅保留 OpenCode 成本映射 ✅
3. **前端收敛**
   - `EngineKind` 类型仅保留 `OPENCODE` ✅
   - 引擎下拉选项仅保留 OpenCode ✅
   - 设置页仅展示 OpenCode 相关配置 ✅
4. **验收标准**
   - UI 不再出现其他引擎 ✅
   - 提交任务、重试、队列执行均使用 OpenCode ✅
   - 双路监督不再硬编码 Codex ✅

## 阶段 1：核心主链路补强 ✅ 已完成

当前项目已有主链路基础能力，本阶段做“差距补齐”而非重写。

1. **固化边界**
   - 请求/响应能力走 `@KremaCommand`
   - 流式进度、状态变化走事件
   - DTO 显式可序列化，命令参数优先使用 request DTO（减少长参数列表）
   - 事件监听必须有解绑
2. **补齐主链能力**
   - **会话管理、历史恢复** ✅
   - **Prompt 提交、批量、插队、编辑、取消、重试、复制、重排** ✅
   - **成本统计与预算阈值** ✅
   - **设置加载/持久化与引擎可用性状态** ✅
   - **工作目录和上下文文件附加** ✅
3. **引擎抽象治理**
   - 统一 timeout、cancel、availability、error surface ✅
   - 统一执行结果结构：`content/tokens/cost/duration` ✅
   - 统一失败可读错误文案（给用户而非只给日志） ✅
4. **验收标准**
   - 核心链路在 dev/prod 均可跑通 ✅（除部分未完成功能外）
   - 队列与消息流状态一致，无明显竞态 ✅
   - 异常时用户可理解、可恢复 ✅

## 阶段 3：MCP 管理系统（先配置层） ✅ 部分完成

先做 registry 管理，不先做完整协议执行。

1. **注册表**
   - 设计 `mcp-registry.json`
   - 字段：`id/name/spec/enabled/targetApps`
   - 可选：`description/tags`
2. **Java 服务能力**
   - 读取/写入注册表
   - 校验 server spec
   - 启用/禁用/删除
   - 导入已有 MCP 配置
3. **OpenCode 同步能力**
   - 可从统一注册表写入 OpenCode 配置
   - 可从 OpenCode 配置反向导入注册表
   - 外部新增配置可发现并同步
   - 禁用不丢元数据
4. **验收标准**
   - 注册表与本地引擎配置可双向同步
    - 同步失败可重试且不破坏已有配置
5. **前端基础交互** ✅
    - 新增/编辑 MCP 服务器弹窗 ✅
    - 基础交互测试 ✅
